# LeeWay Agent Code Generation Law

All code-writing surfaces in this repository are bound to LeeWay construction law.
The LeeWay Application Standards skill is the reusable instruction package that teaches this law to code-writing agents.

## Learned Construction Behavior

All agents that produce, modify, repair, cleanse, verify, or package LeeWay code must begin from LeeWay identity. This includes Agent Lee, Codex, autocomplete systems, local model hive participants, MCP agents, repair scripts, and future LLM-based coding tools.

No generated code is valid unless it includes the necessary LeeWay identity, classification, graph coverage, and verification pathway at creation time.

Before code is proposed or written, the agent must identify the LeeWay domain, pipeline, node, owner, touched-file classification, and verification path.

No LeeWay action is valid unless the actor, prompt, intent, transaction, node, authority, gate, evidence, and receipt are all identifiable and traceable.

No LLM output may become LeeWay action until it has passed through a LeeWay Agent transaction with actor ID, prompt ID, intent ID, authority check, graph mapping, gate verification, evidence, and receipt.

No object may become active unless it has LeeWay identity pulse coverage with origin status, timestamps, hash, actor lineage, transaction lineage, verification evidence, and receipt.

## Bound surfaces

- Codex
- ChatGPT-style agents
- autocomplete systems
- MCP and plugin-routed agents
- local model hive generation
- the Agent Lee engineering loop

## Mandatory conditions

No code output is complete unless:

- every active file has LeeWay metadata
- every active node has identity graph coverage
- every command and event has an owner path
- every pipeline has one active source of truth
- every generated artifact is classified
- every cleanup plan is category-gated
- every action or attempted action has a LeeWay Tracer Pack when acceptance, rejection, quarantine, conversion, or induction occurs
- every active object has identity pulse coverage
- the identity graph gate passes
- the identity pulse gate passes
- the identity mesh gate passes
- the tracer pack gate passes
- the construction law gate passes
- the application integrity gate passes
