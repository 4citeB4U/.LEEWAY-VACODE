# LeeWay Identity Mesh Law

LeeWay is not a library of standards. LeeWay is the operating physics of the agentic hive.

No LeeWay ID is valid unless it contains enough source, actor, intent, transaction, classification, ownership, and verification data to make the action traceable back to its origin.

The ID string is the address. The identity record is the truth.

## Required ID Families

- `LEEWAY_APP`
- `LEEWAY_ACTOR`
- `LEEWAY_INTENT`
- `LEEWAY_PROMPT`
- `LEEWAY_TX`
- `LEEWAY_GATE`
- `LEEWAY_EVENT`
- `LEEWAY_CMD`
- `LEEWAY_FILE`
- `LEEWAY_ARTIFACT`
- `LEEWAY_EVIDENCE`
- `LEEWAY_RECEIPT`
- `LEEWAY_ATTEST`
- `LEEWAY_POLICY`
- `LEEWAY_AUTHORITY`
- `LEEWAY_QUARANTINE`
- `LEEWAY_TRACE`

## Record Law

Every LeeWay identity record must preserve:

- actor ID
- prompt ID
- intent ID
- transaction ID
- authority ID
- node ID
- policy or gate ID when applicable
- evidence ID
- receipt ID
- classification
- owner
- verification path

No LeeWay action is valid unless the actor, prompt, intent, transaction, node, authority, gate, evidence, and receipt are all identifiable and traceable.
