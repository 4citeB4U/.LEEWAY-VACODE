# LeeWay Application Construction Law

LeeWay Standard is the default construction law for Agent Lee.

If any agent, autocomplete system, LLM, tool, command, script, or repair process writes LeeWay code, it must write in LeeWay form from the first draft.

## Core law

- No active code without a LeeWay ID.
- No pipeline without a named LeeWay owner.
- No command without an emitted and handled route.
- No event without a source, consumer, and verification path.
- No file without classification.
- No runtime path without a single source of truth.
- No fallback without explicit fallback classification.
- No generated artifact in production unless classified and verified.
- No patch over duplicate code when removal or quarantine is the correct repair.
- No feature is complete until the identity graph gate and the application integrity gate pass.

## Required closure

Every valid LeeWay change must prove:

- the code works
- the application still works
- every file has classification
- every node has an ID
- every route has an owner
- every generated artifact is classified
- evidence and receipts were written
