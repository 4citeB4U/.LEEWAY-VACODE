# LeeWay Application Construction Law

LeeWay Standard is the default construction law for Agent Lee.

If any agent, autocomplete system, LLM, tool, command, script, or repair process writes LeeWay code, it must write in LeeWay form from the first draft.

No LeeWay action is valid unless the actor, prompt, intent, transaction, node, authority, gate, evidence, and receipt are all identifiable and traceable.

No LLM output may become LeeWay action until it has passed through a LeeWay Agent transaction with actor ID, prompt ID, intent ID, authority check, graph mapping, gate verification, evidence, and receipt.

## Core law

- No active code without a LeeWay ID.
- No pipeline without a named LeeWay owner.
- No command without an emitted and handled route.
- No event without a source, consumer, and verification path.
- No file without classification.
- No runtime path without a single source of truth.
- No fallback without explicit fallback classification.
- No generated artifact in production unless classified and verified.
- No patch over duplicate code when removal or quarantine is the correct repair.
- No action, rejection, quarantine, conversion, or agent induction without a LeeWay Tracer Pack.
- No object without identity pulse, origin status, hash, lineage, verification path, and receipt.
- No feature is complete until the identity graph gate and the application integrity gate pass.

## Permanent construction sequence

1. Identify the LeeWay domain.
2. Identify the pipeline.
3. Identify the node.
4. Assign or reuse the LeeWay ID.
5. Classify every touched file.
6. Declare the owner.
7. Declare inputs and outputs.
8. Declare commands and events emitted and handled.
9. Update the identity graph.
10. Avoid duplicate runtime paths.
11. Remove or quarantine stale paths instead of patching around them.
12. Run the construction law gate.
13. Run the identity graph gate.
14. Run the identity pulse gate.
15. Run the application integrity gate.
16. Write evidence and receipt.

## Required closure

Every valid LeeWay change must prove:

- the code works
- the application still works
- every file has classification
- every node has an ID
- every route has an owner
- every generated artifact is classified
- evidence and receipts were written
